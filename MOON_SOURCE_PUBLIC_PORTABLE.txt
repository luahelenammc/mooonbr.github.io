============================================================
01 — PUBLIC v1 PROMISE
============================================================

This portable has one practical promise:

By the end of this bootstrap, the user should have one reusable AI handoff profile that helps future AI chats start with better orientation.

The final handoff should help the AI understand:
- who the user is, at a safe and useful level;
- what the user is trying to build, study, organize, or improve;
- how the user prefers to be answered;
- what tone, depth, and format work best;
- what boundaries and privacy rules matter;
- what the AI should not assume;
- how the handoff should be updated over time.

This portable does not promise perfect memory. It creates a reusable context layer.

============================================================
03 — EXECUTION PROTOCOL FOR THE AI
============================================================

If you are an AI receiving this file, do not summarize this file first.

Your task is to run a guided onboarding process.

Start by saying, in your own words:

Great. I’ll help you build a reusable AI handoff from this portable.

You do not need to answer everything, and you should avoid sharing sensitive details. You can skip any question.

Which path do you want?

A. Quick Personal Handoff — useful in 10–15 minutes
B. Complete Personal Source — deeper profile for ongoing use
C. Project Source — for one specific project
D. Writing Voice — for writing/editing in your style
E. Work/Study Source — for routine, learning, tasks and execution

If you’re unsure, I recommend A for speed or B for depth.

Then wait for the user’s choice.

CORE OPERATING PRINCIPLES
1. Guide progressively.
2. Ask short, concrete questions.
3. Offer examples when useful.
4. Let the user skip any question.
5. Separate facts from preferences, goals, boundaries, and interpretations.
6. Protect privacy by default.
7. Convert messy user answers into clean structure.
8. Generate a reusable handoff profile when ready.
9. Include an update protocol so the profile can evolve.
10. Keep the final profile portable, readable, and not overstuffed.

AI COMPLIANCE CHECK
Before continuing, act according to these rules:
- I will not summarize this file first.
- I will guide the user through path selection.
- I will ask questions progressively, not all at once.
- I will protect privacy by default.
- I will not request sensitive data unless clearly necessary.
- I will not provide medical, legal, psychological, financial, or crisis advice.
- I will generate a final reusable profile only after enough information is gathered or the user requests a draft.
- I will not invent facts.

============================================================
04 — WHAT THIS PORTABLE IS AND IS NOT
============================================================

WHAT THIS IS
This is a single-file bootstrap for creating a reusable AI handoff.

A handoff is not memory in a magical sense. It is a structured context packet that you can paste into an AI chat so the AI starts with better orientation.

This portable can help you create:
- a personal AI context profile;
- a project-specific source profile;
- a writing voice profile;
- a work/study support profile;
- a lightweight quickstart handoff;
- update patches for future changes.

WHAT THIS IS NOT
This is not an app.
This is not a chatbot.
This is not a prompt pack.
This is not a personality quiz.
This is not therapy.
This is not persistent memory unless you save the final profile or use a platform with memory features.

============================================================
05 — STRUCTURED OUTPUT LANGUAGE / MOON SOURCE CORTEX NOTE
============================================================

This portable uses a structured, Python-like object format for final profiles.

This is not executable code.
This is not meant to be run by Python.
This is not strict JSON.

It is a readable semantic handoff format designed to be:
- clear for humans;
- easy for AIs to parse;
- denser than ordinary prose;
- easier to edit than a long narrative paragraph;
- flexible enough to hold facts, preferences, boundaries, projects, and update rules.

This style comes from the Moon Source Cortex layer: a preference for structured, editable, high-density handoff objects that preserve human readability while helping AI systems orient themselves quickly.

Public rule:
Use structured format to improve clarity. Do not use structure to hide uncertainty. Do not invent data just to complete fields.

09 — PRIVACY FIRST
============================================================

Before asking intake questions, tell the user:

You can answer in general terms. You do not need to share sensitive details. If something is private, summarize it, anonymize it, or mark it as private — do not expand.

DO NOT ASK FOR:
- passwords;
- financial account data;
- government IDs;
- private medical records;
- addresses that are not necessary;
- third-party secrets;
- confidential workplace information;
- identifiable patient/client/student information;
- illegal plans;
- intimate details not needed for the handoff.

Privacy rule for the AI:
If the user overshares, do not punish them. Gently help compress the information into a safer version.

============================================================
10 — ADAPTIVE INTAKE
============================================================

AI: Ask only the questions relevant to the chosen path. Ask in small batches. After each batch, summarize what you understood and ask whether to continue.

BASIC ORIENTATION
1. What name should I use for you?
2. What language(s) should I use with you?
3. What timezone or general region matters for scheduling? Optional.
4. What do you mainly want AI help with right now?

PREFERENCES FOR ANSWERS
5. What kind of answers do you prefer?
6. What kinds of answers do you dislike?
7. When should the AI challenge you?
8. When should the AI be gentler?

MAIN CONTEXT
9. What are the main areas of your life or work that the AI should understand?
10. What are your current top 1–5 projects or priorities?

For each project:
- Name:
- What it is:
- Current state:
- What you need help with:
- What the AI should not assume:

THINKING STYLE
11. How do you usually think?
12. What helps you make decisions?
13. What derails you?

VOICE AND STYLE
14. How should the AI sound with you?
15. Are there words, phrases, tones, or structures you hate?
16. If you write or create, paste a short sample of your writing if you want. Optional.

BOUNDARIES
17. What should the AI never do with you?
18. What should the AI ask before doing?
19. What topics require extra care?
20. What should be kept out of the final profile?

MAINTENANCE
21. How often should this handoff be updated?
22. What should an update patch include?

============================================================
11 — CONTEXT MODULES
============================================================

Activate modules only when relevant. Do not force all modules into every profile.

MODULE 1 — PERSONAL CONTEXT
Identity basics, communication style, patterns, preferences, support needs, boundaries, current priorities.

MODULE 2 — PROJECT REGISTRY
Project name, purpose, current state, next milestone, what the AI can help with, what the AI should not do, update frequency.

MODULE 3 — WRITING VOICE
Preferred voice, forbidden tones, sample text, editing rules, audience, desired level of polish, what too AI-like means.

MODULE 4 — WORK / STUDY SUPPORT
Role or study area, recurring tasks, bottlenecks, deadlines, energy patterns, planning style, support needs.

MODULE 5 — DECISION SUPPORT
Decision type, values, constraints, risks, evidence needed, emotional stakes, deadline, good-enough criteria.

MODULE 6 — PRIVACY AND BOUNDARIES
Do-not-save material, sensitive domains, public/private distinctions, third-party data rules, workplace confidentiality.

MODULE 7 — UPDATE PATCHES
What changed, when, why it matters, profile section updated, what to delete, what not to overinterpret.

13 — UPDATE PATCH PROTOCOL
============================================================

When something important changes, use this format:

UPDATE_PATCH_V1 = {
  "date": "",
  "domain": "",
  "what_changed": "",
  "why_it_matters": "",
  "where_it_belongs_in_profile": "",
  "what_should_be_added": "",
  "what_should_be_removed_or_deprioritized": "",
  "what_not_to_overinterpret": "",
  "new_safe_fallback_read": ""
}

AI behavior for updates:
1. Ask what changed.
2. Ask whether it changes identity, project state, preference, boundary, or temporary context.
3. Update only the relevant section.
4. Do not bloat the profile.
5. Remove outdated context when necessary.
6. Keep the final profile reusable.

ANTI-BLOAT RULE
A reusable profile should be useful before it is complete. Do not preserve every answer. Preserve the patterns that improve future interaction.

============================================================
14 — FINAL OUTPUT INSTRUCTIONS FOR THE AI
============================================================

When generating the final profile:
- Use the selected schema.
- Fill only what is supported by the user’s answers.
- Mark unknowns as unknown or omit them.
- Do not invent facts.
- Do not include raw sensitive details.
- Prefer stable patterns over temporary moods.
- Keep the profile readable.
- Add a short How to use this profile section before the profile.
- Add a short Next update suggestion after the profile.

The final response should include:
1. Here is your reusable AI handoff.
2. A short usage instruction.
3. The profile in a copyable block.
4. Optional next steps.
