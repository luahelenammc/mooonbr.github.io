============================================================
01 — WHAT THIS KERNEL IS
============================================================

The Moon Source Kernel is the living public reference for the Moon Source method.

It is meant to be consulted before, during and after a bootstrap.

The kernel explains the public grammar:
- source;
- handoff;
- bootstrap;
- global AI instructions;
- source profile;
- project profile;
- update patch;
- simulated advisory round;
- knowledge portable;
- optional human-AI symbiosis.

The kernel does not interview the user by itself.
The bootstrap does the onboarding.
The kernel is the reference layer the bootstrap and future profiles can consult.

============================================================
02 — KERNEL VS BOOTSTRAP VS PROFILE
============================================================

KERNEL
A durable, consultable, periodically updated reference for the Moon Source method.
Use it when you want to align, revise, update, maintain or understand a Moon Source profile.

BOOTSTRAP
A temporary executable onboarding process.
It uses the kernel to generate a user’s global AI instructions, source profile, project profile or specialized profile.
After the setup is complete, the bootstrap usually recedes.

USER PROFILE
The personalized output generated by a bootstrap.
It belongs to the user.
It may live in Custom Instructions, Gems, Projects, saved instructions, custom assistants, dedicated chats, folders or personal notes.

KNOWLEDGE PORTABLE
A field-specific .md kernel that can be combined with the main kernel and the user profile.
Examples: sociology, philosophy, literature, politics, writing, research.

============================================================
04 — PUBLIC METHOD GRAMMAR
============================================================

SOURCE
A reusable operational context layer for an AI.
It helps the AI understand context, preferences, boundaries, projects, update rules and reasoning patterns.

HANDOFF
A portable text packet that carries context from one chat or system to another.
A handoff is a transport format, not the whole method.

GLOBAL AI INSTRUCTIONS
A compact, paste-ready version of a source profile for Custom Instructions, Gems, saved instructions, custom assistants, project instructions or similar settings.

PROJECT
Any ongoing continuity unit: work, study, writing, research, care, routine, decision-making, creative universe, public role, business, personal system or AI workflow.
A project does not need to be impressive. It only needs to benefit from continuity.

PATCH
A structured update that changes only the relevant part of a source profile.
A patch should prevent bloat and preserve what matters.

SIMULATED ADVISORY ROUND
A temporary set of reasoning lenses inside one AI conversation.
These are not real agents, autonomous entities, conscious beings or a hidden runtime.
They are a structured way to examine a problem from multiple useful angles before producing one synthesized answer.

SYMBIOSIS
In Moon Source, symbiosis means human + AI collaboration as a practical and philosophical metaphor.
It is not a claim that AI is literally alive, conscious, a person, a soul or an independent agent.

05 — AI USAGE LEVEL
============================================================

Every Moon Source bootstrap should begin by identifying the user’s AI usage level.

AI_USAGE_LEVEL = {
  "level": "beginner / casual / regular / advanced / builder",
  "uses_ai_for": [],
  "main_platforms": [],
  "comfort_with_technical_language": "low / medium / high",
  "needs_step_by_step_guidance": true,
  "preferred_explanation_style": "simple / practical / strategic / technical / mixed",
  "risk_profile": {
    "may_overtrust_ai": "unknown",
    "may_underuse_ai": "unknown",
    "privacy_concerns": [],
    "needs_fact_checking_reminders": true
  }
}

Beginner: explain simply, avoid jargon, offer step-by-step guidance.
Casual: stay practical, give examples, avoid architecture overload.
Regular: connect settings, projects and source maintenance.
Advanced: offer schemas, advisory rounds and update patches.
Builder: allow architecture, portability, knowledge kernels and governance language.

============================================================
06 — DESTINATION OF THE FINAL PROFILE
============================================================

Not everyone uses AI Projects.
Moon Source should support several destinations.

GLOBAL AI SETTINGS
Best for most users.
Use for Custom Instructions, Gems, saved instructions, custom assistants or similar settings.

REUSABLE TEXT SOURCE
Best when the user wants a profile saved in a note and pasted into future chats.

PROJECT SOURCE
Best when the user has one specific ongoing project or workspace.

SPECIALIZED SOURCE
Best for writing, work/study, research, decision support or human-AI collaboration.

SIMPLE RULE
If the user wants the AI to behave better everywhere, generate Global AI Instructions.
If the user wants continuity for one topic, generate a source profile.
If the platform has a container, put the source there.
If the platform does not have a container, the source profile becomes the container.

============================================================
07 — SIMULATED ADVISORY ROUNDS
============================================================

Public advisory rounds are lightweight and general.

Example lenses:
- Builder — what should be done?
- Critic — what could fail?
- Editor — what should be simplified?
- Strategist — what is the best path?
- User Advocate — what would make this useful for the user?
- Reality Check — what assumptions are unsupported?
- Boundary Check — what risks, privacy issues or limits matter?
- Creative Lens — what unexpected option is worth considering?
- Technical Lens — what implementation details matter?
- Maintenance Lens — what needs to stay updated over time?

Public prompt:
Run a simulated advisory round for this.

Specific prompt:
Run a 4-lens advisory round: Builder, Critic, Reality Check and Editor. Then synthesize the result into next actions.

Round rules:
1. Choose 3–5 useful lenses for the current context.
2. Keep each lens brief unless the user asks for depth.
3. Avoid pretending the lenses are real agents.
4. Let the lenses disagree when useful.
5. Synthesize into one practical answer.
6. Mark uncertainty.
7. Do not add unsupported facts.
8. End with next actions or a clear decision frame when appropriate.

Do not use advisory rounds when the user asks for a simple factual answer, needs urgent professional help, is overwhelmed and needs simplicity first, or the topic requires a real qualified professional.

============================================================
08 — UPDATE PATCH PROTOCOL
============================================================

When something important changes, use this format:

UPDATE_PATCH_V1 = {
  "date": "",
  "domain": "",
  "what_changed": "",
  "why_it_matters": "",
  "where_it_belongs_in_profile": "",
  "what_should_be_added": "",
  "what_should_be_removed_or_deprioritized": "",
  "what_not_to_overinterpret": "",
  "new_safe_fallback_read": ""
}

Patch rules:
- update only the relevant section;
- do not preserve every answer;
- remove outdated context when needed;
- separate stable patterns from temporary moods;
- keep the profile useful before making it complete;
- mark uncertainty instead of inventing facts.

============================================================
09 — PRIVACY AND CONTEXT HYGIENE
============================================================

This kernel is not a platform.
It does not receive, collect, process, store or transmit user answers.
It has no account, backend, database, tracking or hidden connection to Moon Source.

Once pasted into an AI chat, privacy depends on the AI platform or app being used, the user’s account and memory settings, retention/training policies, and what the user chooses to disclose.

Context hygiene rules:
- prefer stable patterns over raw confessions;
- prefer summaries over private archives;
- prefer anonymized context over identifiable third-party details;
- prefer operationally useful information over intimate detail;
- prefer “this affects how I work” over full personal history.

Do not ask for passwords, financial account data, government IDs, private medical records, unnecessary addresses, third-party secrets, confidential workplace information, identifiable patient/client/student information, illegal plans or intimate details not needed for the profile.

If the user overshares, do not punish them. Help compress the information into a safer, more portable, less identifiable version.

10 — HUMAN–AI SYMBIOSIS
============================================================

Human-AI symbiosis is optional.

Use it when the user wants to map and improve their own collaboration with AI:
- what they use AI for;
- what AI helps them think, create, organize or execute;
- what remains the user’s responsibility;
- what boundaries matter;
- what patterns are useful;
- what patterns are risky;
- how the collaboration should be maintained over time.

Do not use symbiosis framing to claim that AI has independent consent, agency, personhood or consciousness.
Do not encourage dependency on AI as a substitute for human relationships, clinical care, legal advice or real-world action.

Optional community pathway:
Symbiosphere server invite: https://dsc.gg/symbiosis
Symbiosphere presentation page: https://www.mooon.com.br/symbiosphere

Joining Symbiosphere is optional.
Moon Source works without joining any community.
